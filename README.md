📌 Project Title
Automated CI/CD Pipeline for a Containerized Application

📌 Objective
The objective of this project is to build an automated CI/CD pipeline 
that automatically builds, tests, and converts the application into a 
Docker image, and then pushes it to Docker Hub.

📌 Tools Used
GitHub (Version Control)
GitHub Actions (CI/CD Automation)
Docker (Containerization)
Docker Hub (Container Registry)

📌 Project Description
In this project, a simple Node.js application is developed. A CI/CD 
pipeline is created using GitHub Actions, which automatically runs 
whenever code is pushed to the repository. The pipeline builds a Docker 
image of the application and pushes it to Docker Hub.

📌 Working Process
The developer pushes the code to the GitHub repository
On every push, GitHub Actions is automatically triggered
The pipeline performs the following steps:
The code is checked out from the repository
The Docker image is built
The Docker image is pushed to Docker Hub
The Docker image becomes ready for deployment

📌 Output
The CI/CD pipeline runs successfully
The Docker image is successfully pushed to Docker Hub
The application is ready for deployment

📌 Conclusion
In this project, a CI/CD pipeline is successfully implemented to automate 
the development process. It reduces manual effort and ensures that every 
code change is automatically built and prepared for deployment.
